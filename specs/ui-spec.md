# UI-спецификация

Описание всех экранов и кросс-экранных виджетов минималистичного UI. UI предназначен для визуальной валидации; все действия дублируют API. Документ описывает структуру каждого экрана, его блоки, поля, кнопки и используемые виджеты со ссылкой на функции из каталога функций основного документа.

## Общие принципы UI

**Шапка (header).** Присутствует на всех экранах кроме регистрации/логина. Содержит:
- Логотип / ссылка на экран списка задач.
- Навигационное меню: Задачи, Проекты, Команды, События.
- Виджет статуса тарифа (правая часть шапки).
- Виджет приглашений и уведомлений (правая часть шапки).
- Меню профиля (аватар) — переход на экран профиля, логаут.

**Структура экранов.** Все экраны состоят из шапки, основной области и, где применимо, боковой панели фильтров.

**Сообщения об ошибках.** Валидационные и доменные ошибки отображаются inline у соответствующих полей; серверные ошибки — тост в правом нижнем углу.

**Подтверждения деструктивных операций.** Удаление задачи, удаление проекта, удаление команды, отзыв прямого доступа, удаление участника — модалка подтверждения «Да / Отмена». Открепление задачи от последнего проекта при отсутствии R-8 подтверждения не требует (согласно принятой модели).

## 1. Экран регистрации

**Назначение.** Создание нового аккаунта.

**Блоки:**
- Заголовок «Регистрация».
- Форма:
  - Поле «Email» (email-валидация).
  - Поле «Пароль» (с требованиями к сложности).
  - Поле «Повторите пароль».
  - Кнопка «Зарегистрироваться».
- Ссылка «Уже есть аккаунт? Войти» → переход на экран логина.

**Функции:** Регистрация.

## 2. Экран логина

**Назначение.** Вход в систему.

**Блоки:**
- Заголовок «Вход».
- Форма:
  - Поле «Email».
  - Поле «Пароль».
  - Кнопка «Войти».
- Ссылка «Забыли пароль?» → экран восстановления пароля.
- Ссылка «Нет аккаунта? Зарегистрироваться» → экран регистрации.

**Функции:** Логин.

## 3. Экран восстановления пароля

**Назначение.** Инициирование сброса пароля по email.

**Блоки:**
- Заголовок «Восстановление пароля».
- Форма:
  - Поле «Email».
  - Кнопка «Отправить ссылку».
- После отправки — информационное сообщение «Если указанный email зарегистрирован, на него отправлена ссылка для сброса пароля».
- На отдельном sub-экране (по ссылке из письма):
  - Поле «Новый пароль».
  - Поле «Повторите новый пароль».
  - Кнопка «Сохранить».

**Функции:** Смена пароля.

## 3a. Экран подтверждения email

**Назначение.** Активация managed-пользователя по ссылке из письма, пришедшего в момент создания managed родителем.

**Блоки:**
- Заголовок «Подтверждение email».
- При валидном токене: сообщение «Ваш email подтверждён. Теперь вы можете войти в систему.» + кнопка «Войти» → экран логина.
- При невалидном или истёкшем токене: сообщение с предложением связаться с администратором, создавшим аккаунт.

**Функции:** Подтверждение email.

## 4. Экран профиля пользователя

**Назначение.** Управление собственными настройками, тарифом, PAT.

**Блоки:**

**Блок «Профиль»:**
- Email (read-only).
- Селектор «Тема» (light/dark).
- Селектор «Язык».
- Кнопка «Сменить пароль» → открывает модалку с полями «Текущий пароль», «Новый пароль», «Повторите».
- Кнопка «Выйти».

**Блок «Тариф»:**
- Текущий тариф (Free/Pro/Team/Enterprise).
- Период подписки (monthly/yearly) и дата окончания текущего периода.
- Текущее использование лимитов — несколько прогресс-баров:
  - Задачи (N / лимит или N / ∞).
  - Проекты владельца.
  - Команды владельца.
  - Автоматизации (макс. в одном проекте из числа владельческих).
  - Участники (самый заполненный проект/команда).
- Состояние банка дней — таблица слоёв: тариф, период, остаток дней, порядок.
- Для Enterprise — текущее число слотов и планируемое уменьшение (если есть).
- Кнопка «Сменить тариф» → переход на экран тарифов.
- Ссылка «История смен тарифа» → модалка/подраздел с таблицей смен.
- Если есть запланированный даунгрейд — плашка «Запланирован даунгрейд на <тариф> с <дата>» и кнопка «Отменить даунгрейд».
- Если есть запланированное уменьшение слотов — плашка с кнопкой «Отменить уменьшение».
- Ссылка «Заблокированные сущности» → экран заблокированных сущностей (виден, если есть хоть одна).

**Блок «Приватные ключи (PAT)»:**
- Таблица PAT: имя, значение токена (полное, видимое владельцу), дата создания, срок истечения (если есть), статус (активен / отозван / истёк).
- Кнопка «Создать PAT» → модалка: поле «Имя», поле «Срок истечения» (опционально, datepicker), кнопка «Создать». После создания — токен виден в таблице.
- По каждому активному PAT — кнопка «Отозвать» с подтверждением.

**Блок «Managed-пользователи»** (виден только обычным пользователям, у managed скрыт):
- Индикатор использования: «N managed-пользователей из M по тарифу».
- Таблица managed: email, статус («Ожидает подтверждения email» / «Активен» / «Деактивирован»), дата создания, действия.
- Кнопка «Создать managed» → модалка: поля «Email», «Пароль», кнопка «Создать». После создания — пояснение «На указанный email отправлено письмо с подтверждением. До подтверждения managed не сможет войти в систему».
- По каждой строке таблицы — кнопки действий:
  - Для ожидающих подтверждения: «Переотправить письмо», «Удалить».
  - Для активных: «Деактивировать», «Сбросить пароль», «Удалить».
  - Для деактивированных: «Ре-активировать», «Удалить».
- Удаление managed — с подтверждением «Все проекты и команды, созданные managed, принадлежат вам и сохранятся. Все его PAT и автоматизации будут удалены».

**Блок «Аккаунт» (для managed-пользователя вместо блока «Тариф» и «Managed-пользователи»):**
- Пометка «Ваш аккаунт управляется пользователем <email родителя>».
- Пояснение, что managed не может менять тариф, передавать владение или менять email.

**Блок «Удаление аккаунта»:**
- Кнопка «Удалить аккаунт» (для всех типов пользователей) — открывает модалку подтверждения с детальным предупреждением:
  - «Все ваши проекты (N), команды (M), managed-пользователи (K), автоматизации, секреты, PAT будут удалены безвозвратно.»
  - «Все задачи, где вы указаны как автор или исполнитель, сохранятся, но ссылки на вас станут "удалённый пользователь".»
  - Поле ввода «Для подтверждения введите свой email».
  - Кнопка «Удалить навсегда».

**Функции:**
- Получить свой профиль.
- Смена темы пользователя.
- Смена языка пользователя.
- Смена пароля.
- Логаут.
- Создать PAT.
- Получить список своих PAT.
- Отозвать PAT.
- Получить текущий тариф пользователя и состояние банка дней.
- Получить историю смен тарифа.
- Получить текущее использование лимитов.
- Отменить запланированный даунгрейд.
- Отменить запланированное уменьшение слотов.
- Создать managed-пользователя.
- Получить список своих managed-пользователей.
- Деактивировать / ре-активировать managed-пользователя.
- Сбросить пароль managed-пользователя.
- Удалить managed-пользователя.
- Удалить свой аккаунт.

**Используемые виджеты:** виджет статуса тарифа (в шапке), виджет приглашений и уведомлений (в шапке).

## 5. Экран списка задач

**Назначение.** Просмотр и фильтрация задач, доступных пользователю. Точка входа в массовые операции.

**Блоки:**

**Панель действий (верхняя):**
- Кнопка «Создать задачу» → модалка создания (см. ниже).
- Кнопка «Массовое действие по фильтру» — активна, когда задан RSQL-фильтр. Открывает модалку выбора операции и параметров.
- Кнопка «Массовое действие по выбранным» — активна, когда отмечены задачи чекбоксами. Открывает модалку выбора операции.

**Панель фильтра:**
- Виджет RSQL-фильтра для построения выражения.
- Счётчик «Найдено: N задач».
- Кнопка «Сбросить».

**Таблица задач** (по умолчанию — все задачи, доступные пользователю):
- Чекбокс выделения (шапка: «выделить все на странице»).
- Колонки: `id` (короткий), `title` (кликабельно → карточка задачи), `status` (бейдж), `assignee` (email + аватар), `projects` (чипы), `tags` (чипы), `updated_at`.
- Пагинация (cursor-based).
- Сортировка по `created_at` / `updated_at` / `title`.

**Модалка «Создать задачу»:**
- Поле «Название» (обязательное).
- Поле «Описание» (markdown-редактор, опционально).
- Селектор «Статус» (default `opened`).
- Виджет поиска пользователя «Ответственный» (опционально).
- Мультиселект проектов (опционально).
- Поле «Теги» (ввод через запятую/пробел).
- Кнопка «Создать».

**Модалка «Массовое действие»:**
- Отображение количества задач, на которые повлияет действие.
- Селектор операции:
  - Изменить статус.
  - Назначить / снять assignee.
  - Добавить / удалить теги.
  - Добавить / убрать блокеры.
  - Прикрепить / открепить от проекта.
  - Выдать / отозвать прямой доступ.
  - Удалить задачи.
- В зависимости от выбранной операции — соответствующие поля ввода.
- Кнопка «Выполнить».
- При ошибке — отображение списка задач, на которых операция не прошла, с причинами (all-or-nothing: ничего не применилось).

**Функции:**
- Получить список задач (с RSQL-фильтром и пагинацией).
- Создать задачу.
- Массовое действие над задачами по RSQL.
- Массовое действие над задачами по списку ID.

**Используемые виджеты:**
- Виджет RSQL-фильтра.
- Виджет поиска пользователя (в создании задачи и в массовых операциях с assignee).
- Виджет выбора прав R-1…R-8 (в массовой операции выдачи доступа).

## 6. Экран карточки задачи

**Назначение.** Просмотр и редактирование одной задачи.

**Блоки:**

**Шапка карточки:**
- `id` задачи (копируемый).
- Поле `title` (inline-редактируемое, если есть R-1).
- Кнопка «Удалить задачу» (если есть R-8) — с подтверждением.

**Основное содержимое:**
- Поле `description` (markdown-редактор, если есть R-2; иначе просмотр).
- Селектор `status` (`opened`/`progress`/`closed`; редактируется при R-6).
- Блок «Ответственный»: виджет поиска пользователя (при R-5); крестик для снятия.
- Блок «Теги»: чипы с крестиками (удаление при R-3); поле ввода нового тега (при R-3).
- Блок «Прикреплено к проектам»: чипы проектов (крестики удаляют прикрепление — требует R-12 в соответствующем проекте); кнопка «Прикрепить к проекту» (поиск проекта, из которых пользователь может выбрать — где у него есть R-12).

**Блок «Блокеры» (редактируется при R-4):**
- Список блокирующих задач (название + статус).
- Кнопка «Добавить блокер» — поиск по задачам.
- Отдельно (read-only): список задач, которые блокирует данная (`blocked_tasks`).

**Блок «Прямые доступы» (виден при R-7):**
- Таблица: получатель (пользователь или команда), выданные права (чипы R-1…R-8), автор выдачи, дата.
- По каждой строке: кнопки «Изменить» и «Отозвать».
- Кнопка «Выдать доступ» — модалка: выбор получателя (пользователь или команда через соответствующий виджет поиска) + виджет выбора прав R-1…R-8 (с маской — не больше, чем есть у текущего пользователя) + кнопка «Выдать».

**Блок «Мета»:**
- Автор (read-only).
- Дата создания, дата последнего изменения.

**Функции:**
- Получить задачу по ID.
- Изменить название задачи.
- Изменить описание задачи.
- Изменить статус задачи.
- Назначить / снять assignee.
- Добавить / удалить теги.
- Добавить / удалить блокеры.
- Прикрепить задачу к проекту.
- Открепить задачу от проекта.
- Удалить задачу.
- Выдать прямой доступ.
- Изменить состав прав у прямого доступа.
- Отозвать прямой доступ.
- Получить список прямых доступов задачи.

**Используемые виджеты:**
- Виджет поиска пользователя.
- Виджет поиска команды.
- Виджет выбора прав R-1…R-8.

## 7. Экран списка проектов

**Назначение.** Обзор проектов пользователя.

**Блоки:**
- Кнопка «Создать проект».
- Панель фильтра: строковый поиск по названию, переключатель «Только мои владения».
- Таблица проектов:
  - Название (кликабельно → карточка проекта).
  - Владелец (email).
  - Моя роль (список прав-чипов в проекте).
  - Число задач в проекте.
  - Статус: «Активен» / «Заблокирован тарифом» (плашка-индикатор).
  - Дата создания.

**Модалка «Создать проект»:**
- Поле «Название».
- Кнопка «Создать». Создатель автоматически становится владельцем с полным набором R-1…R-14.

**Функции:**
- Получить список проектов.
- Создать проект.

## 8. Экран карточки проекта

**Назначение.** Управление проектом, его участниками, автоматизациями, секретами, импортом.

**Блоки:**

**Шапка:**
- Поле `name` (inline-редактируемое при R-9).
- Плашка «Заблокирован тарифом», если применимо.
- Кнопка «Передать владение» (для владельца) → модалка с поиском пользователя и подтверждением.
- Кнопка «Удалить проект» (при R-13) — с подтверждением.

**Вкладка «Задачи»:**
- Встроенный список задач проекта (аналог экрана списка задач, но с предустановленным фильтром `projects==<id>`).
- Кнопки «Создать задачу (в этом проекте)», «Импортировать CSV» (при R-14).

**Модалка «Импорт CSV» (при R-14):**
- Кнопка загрузки CSV-файла.
- Информационный блок о формате: обязательная колонка `title`, опциональные `description`/`status`/`assignee`/`tags`, разделитель тегов `;`, лимит 10 000 строк / 10 МБ.
- После выбора файла — кнопка «Импортировать».
- После попытки — отчёт: «N задач импортировано» или «Импорт отклонён, ошибки:» с таблицей «строка — колонка — причина».

**Вкладка «Участники» (при R-10):**
- Подраздел «Пользователи»:
  - Таблица: email, чипы прав (14), дата вступления, кнопки «Изменить права», «Удалить».
  - Кнопка «Пригласить пользователя» — модалка с виджетом поиска пользователя и виджетом выбора прав R-1…R-14.
- Подраздел «Команды»:
  - Таблица: название команды, чипы прав, статус «Активна» / «Заморожена в проекте», дата вступления, кнопки «Изменить права», «Удалить».
  - Кнопка «Пригласить команду» — модалка с виджетом поиска команды и виджетом выбора прав.
- Подраздел «Ожидающие приглашения»:
  - Таблица pending-приглашений: получатель (user/team), предложенные права, статус активности («Активное» / «Неактивное — превышен лимит»), автор и дата.

**Вкладка «Автоматизации» (при R-11):**
- Таблица автоматизаций: название, триггер (тип события), условие (сокращённое отображение RSQL), включена ли, счётчик подряд неуспехов, кнопки «Редактировать», «Включить/Выключить», «Удалить».
- Плашка «Автоматически выключена после 10 неуспехов», если применимо, с кнопкой «Включить».
- Кнопка «Создать автоматизацию» → переход на экран автоматизации (создание).

**Вкладка «Секреты» (при R-11):**
- Таблица секретов: ключ (только имя), дата создания, дата последнего изменения, кнопки «Обновить значение», «Удалить».
- Кнопка «Создать секрет» — модалка: поле «Ключ», поле «Значение». Значение нигде не отображается после сохранения.

**Функции:**
- Получить проект.
- Переименовать проект.
- Пригласить пользователя в проект.
- Пригласить команду в проект.
- Изменить состав прав участника проекта.
- Удалить участника из проекта.
- Передать владение проектом (инициировать).
- Импортировать CSV в проект.
- Удалить проект.
- Создать автоматизацию.
- Получить список автоматизаций.
- Удалить автоматизацию.
- Включить / выключить автоматизацию.
- Создать / обновить / удалить секрет.
- Получить список секретов проекта (без значений).
- Создать задачу (в контексте проекта).
- Получить список задач (с предустановленным фильтром по проекту).

**Используемые виджеты:**
- Виджет поиска пользователя.
- Виджет поиска команды.
- Виджет выбора прав (R-1…R-14).
- Виджет RSQL-фильтра (во встроенном списке задач).

## 9. Экран списка команд

**Назначение.** Обзор команд пользователя.

**Блоки:**
- Кнопка «Создать команду».

**Модалка «Создать команду»:** поле «Название», кнопка «Создать». Создатель автоматически становится владельцем и админом.

## 11. Экран автоматизации

**Назначение.** Создание и редактирование одной автоматизации.

**Блоки:**

**Шапка:**
- Поле «Название» (обязательное).
- Переключатель «Включена / Выключена».

**Блок «Триггер»:**
- Селектор «Тип события» — выбор из каталога событий (task.*, project.*, team.*, automation.*, user.*). Группировка по сущности.

**Блок «Условие»:**
- Виджет RSQL-фильтра (опционально).
- Пояснение: «Действие выполняется, если условие истинно для связанной задачи события. Оставьте пустым, чтобы срабатывать на каждое событие».

**Блок «Действие» (вызов API):**
- Селектор «Метод» (GET/POST/PUT/PATCH/DELETE).
- Поле «URL» (поддержка плейсхолдеров `{{…}}` и `{{secrets.<key>}}`).
- Таблица заголовков (ключ–значение, с плейсхолдерами).
- Поле «Тело запроса» (textarea, с плейсхолдерами).
- Блок помощи: список доступных плейсхолдеров для выбранного типа события (например, для `task.status_changed`: `{{task.id}}`, `{{task.title}}`, `{{task.status}}`, `{{event.author}}`, `{{event.created_at}}`) и секретов проекта.

**Блок «Тестирование» (опционально):**
- Кнопка «Выполнить тестовый вызов» — подставляет демо-значения и вызывает API, показывает ответ. Тестовый вызов не генерирует срабатывание.

**Нижняя панель:**
- Кнопка «Сохранить».
- Кнопка «Удалить» (если редактируется существующая) — с подтверждением.

**История срабатываний** (внизу, только для существующей автоматизации):
- Таблица последних N срабатываний: дата, номер попытки, статус (успех/ошибка/таймаут/ретрай), HTTP-код ответа, выдержка из тела ответа, ссылка на связанное событие.

**Функции:**
- Создать автоматизацию.
- Изменить автоматизацию.
- Удалить автоматизацию.
- Включить / выключить автоматизацию.

**Используемые виджеты:** виджет RSQL-фильтра.

## 16. Кросс-экранные виджеты

### 16.1. Виджет поиска пользователя

**Назначение.** Выбор одного или нескольких пользователей системы.

**Структура:**
- Поле ввода с автодополнением (подсказки появляются после 2 символов).
- Запрос ищет по email и (опционально) по части локальной части.
- Выпадающий список результатов: email + аватар.
- При множественном режиме — чипы выбранных пользователей с крестиками.

**Используется:**
- Экран карточки задачи (назначение assignee, выдача прямого доступа).
- Экран карточки проекта (приглашение пользователя, передача владения).
- Экран карточки команды (приглашение пользователя, передача владения).
- Экран списка задач (модалка создания задачи — assignee; модалка массового действия — смена assignee, выдача доступа).
- Экран ленты событий (фильтр по автору).

**Функции:** поиск пользователей.

### 16.2. Виджет поиска команды

**Назначение.** Выбор команды.

**Структура:**
- Поле ввода с автодополнением по названию.
- Выпадающий список: название команды + владелец.
- Мультивыбор через чипы.

**Используется:**
- Экран карточки задачи (выдача прямого доступа команде).
- Экран карточки проекта (приглашение команды).
- Экран списка задач (модалка массового действия — выдача доступа команде).

**Функции:** поиск команд.

### 16.3. Виджет RSQL-фильтра

**Назначение.** Построение RSQL-выражений для фильтрации списков и условий автоматизаций.

**Структура:**
- Основной режим — текстовое поле с подсветкой синтаксиса и inline-валидацией.
- Подсказки автодополнения при вводе: имена полей, операторы, значения enum (для `status`), литерал `null`, функции относительных дат (`now`, `now-7d`).
- Показ синтаксических ошибок inline (подсвечивание позиции, сообщение).
- Счётчик длины (лимит 4096 символов).
- Примеры справа или по кнопке «Помощь» — карточки типовых фильтров: «мои задачи», «без assignee», «закрытые за неделю», «с тегом urgent» и т.п.

**Используется:**
- Экран списка задач (основной фильтр).
- Экран ленты событий — опционально для дополнительного фильтра по связанной задаче (если применимо).
- Экран автоматизации (поле «Условие»).
- Экран карточки проекта (встроенный список задач — если пользователь разворачивает расширенный фильтр).

**Функции:** клиентский компонент, сам по себе API не вызывает. Передаёт RSQL-строку в вызовы API получения списков.

### 16.4. Виджет выбора прав (R-1…R-14)

**Назначение.** Задание набора прав.

**Структура:**
- Две группы чекбоксов: «Права на задачи проекта» (R-1…R-8) и «Права на проект» (R-9…R-14). Для контекста прямого доступа на задачу — только R-1…R-8.
- Каждое право — чекбокс с подписью и подсказкой-тултипом, описывающей действие (например, «R-6. Изменение статуса задачи»).
- При выдаче пользователем другому пользователю — чекбоксы по правам, которыми пользователь не обладает, отключены и затенены, с подсказкой «Нельзя выдать права, которых нет у вас». Это реализует ограничение R-7.
- Быстрые пресеты-кнопки: «Полные права», «Только чтение и комментирование статуса», «Снять все».

**Используется:**
- Экран карточки задачи (модалка выдачи и изменения прямого доступа — только R-1…R-8).
- Экран карточки проекта (модалки приглашения и изменения прав участника — R-1…R-14).
- Экран списка задач (модалка массового действия «Выдать прямой доступ» — R-1…R-8).

**Функции:** клиентский компонент, API не вызывает самостоятельно.

### 16.5. Виджет приглашений и уведомлений

**Назначение.** Индикация и быстрый доступ к входящим событиям, требующим реакции пользователя.

**Структура:**
- Иконка «колокольчик» в шапке с бейджем-счётчиком (сумма pending-приглашений и запланированных изменений).
- При клике — выпадающая панель с секциями:
  - Приглашения в команды (N).
  - Приглашения в проекты (N).
  - Передачи владения (N).
  - Системные уведомления (автовыключение автоматизаций, заморозка команд в проектах, блокировки тарифом).
- Кнопка «Перейти ко всем» → экран входящих приглашений.
- Быстрые действия «Принять/Отклонить» прямо из панели (для приглашений и передач владения).

**Используется:** во всех экранах через шапку.

**Функции:**
- Получить список pending-приглашений (опосредованно, через функции принятия/отклонения и соответствующие сводки).

### 16.6. Виджет статуса тарифа

**Назначение.** Индикация текущего тарифа, использования лимитов и тревог, связанных с тарификацией.

**Структура:**
- Бейдж с названием тарифа в правой части шапки.
- При клике — выпадающая панель:
  - Текущий тариф и дата окончания периода.
  - Мини-прогресс-бары по ключевым лимитам (задачи, проекты, команды).
  - Предупреждения: «Лимит участников в проекте X почти исчерпан», «Запланирован даунгрейд», «Есть заблокированные сущности (N)».
  - Ссылка «Подробнее» → экран профиля (раздел «Тариф») или экран заблокированных сущностей.
- Визуальная окраска бейджа меняется при превышениях и тревогах (жёлтый/красный).

**Используется:** во всех экранах через шапку.

**Функции:**
- Получить текущий тариф и состояние банка дней.
- Получить текущее использование лимитов.
