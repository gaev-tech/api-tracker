Три уровня высоты. Все компоненты принадлежат одному из уровней. Элемент, вложенный в контрол, берётся на уровень ниже.

md 36px 13–14px 16px 12px --radius-md · 8px md-кнопка, md-инпут, хлебные крошки, md-аватар, md-тег

Table · Таблица
Компонент-обёртка для списков с фиксированным горизонтальным отступом 24 px на всех уровнях. Три слоя: .tbl-toolbar — строка поиска + счётчик + действие; .tbl-head — заголовки колонок; .tbl-body — прокручиваемые строки .trow. grid-template-columns задаётся inline и одинаково у .tbl-head и каждой .trow.
Toolbar · .tbl-toolbar
Flex-строка с gap:8px и padding:8px 12px. Поисковая строка растягивается, счётчик — статичный текст, кнопка прижата вправо через margin-left:auto. Поиск и кнопки — высота 36 px (md, без модификатора размера).

Элементы укладываются сверху вниз внутри .tbl. Высота .tbl-body заполняет оставшееся пространство; содержимое прокручивается независимо.

Обзор продукта

API Tracker — трекер задач, спроектированный вокруг API как основного канала взаимодействия. Вся функциональность доступна через задокументированные REST-эндпоинты с аутентификацией по PAT.

API-first
Визуальный интерфейс — вторичный канал. Всё, что можно сделать в UI, можно сделать через API.

Аутентификация

Все запросы аутентифицируются через Personal Access Token (PAT). Передавайте токен в заголовке:

Authorization: Bearer pat_your_token_here

Создать PAT

Токены создаются в настройках профиля. Каждый токен имеет опциональную дату истечения.

POST /pats 
{
  "name": "CI/CD pipeline",
  "expires_at": "2026-12-31T00:00:00Z"
}
Шаблоны

Регистрация и выход через API

Создайте аккаунт, подтвердите email, получите PAT для последующих запросов и завершите сессию.

1. Зарегистрироваться

POST /auth/register 
{
  "email": "ivan@corp.com",
  "password": "••••••••",
  "display_name": "Иван Петров"
}

2. Подтвердить email

POST /auth/verify-email 
{ "token": "<token из письма>" }

3. Создать PAT

POST /pats 
{ "name": "my-token", "expires_at": "2027-01-01T00:00:00Z" }

4. Выйти

DELETE /auth/session 
— тело не требуется —
Шаблоны

Создать задачу с разными полями

Задача может существовать без проекта (личная) или быть прикреплена к одному и более проектам.

Полный набор полей

POST /tasks 
{
  "title":       "Добавить rate limiting в API Gateway",
  "status":      "opened",
  "priority":    "high",
  "assignee_id": "usr_ivan",
  "project_ids": ["proj_backend", "proj_infra"],
  "tags":        ["security", "gateway"],
  "due_at":      "2026-05-01T00:00:00Z"
}
Шаблоны

Найти задачи по RSQL

Все списочные эндпоинты принимают параметр filter в синтаксисе RSQL.

Примеры фильтров

Открытые задачи с высоким приоритетом
GET /tasks?filter=status==opened;priority==high
Задачи в проекте с тегом
GET /tasks?filter=projects==proj_backend;tags==ci-failure
Назначенные конкретному пользователю, созданные после даты
GET /tasks?filter=assignee_id==usr_ivan;created_at=gt=2026-01-01
Шаблоны

Массовое изменение поля нескольких задач

Используйте RSQL-фильтр для выборки и patch-эндпоинт для обновления поля у всех подходящих задач за один запрос.

Закрыть все задачи релиз-кандидата

PATCH /tasks/bulk 
{
  "filter": "projects==proj_backend;tags==release-candidate",
  "patch":  { "status": "closed" }
}

Переназначить исполнителя

PATCH /tasks/bulk 
{
  "filter": "assignee_id==usr_old;status!=closed",
  "patch":  { "assignee_id": "usr_new" }
}
Шаблоны

Онбординг RnD-команды

Создайте управляемых пользователей, проект, команду и стартовые задачи в одном сценарии — полная настройка рабочего пространства через API.

1. Создать управляемых пользователей

POST /managed-users 
[
  { "email": "alice@corp.com", "display_name": "Alice" },
  { "email": "bob@corp.com",   "display_name": "Bob" }
]

2. Создать проект

POST /projects 
{ "name": "RnD Q2", "description": "Research & Development Q2 2026" }

3. Создать команду и добавить участников

POST /teams 
{
  "name":       "RnD Team",
  "project_id": "proj_rnd_q2",
  "members":    ["usr_alice", "usr_bob"]
}

4. Создать стартовые задачи

POST /tasks/bulk-create 
[
  { "title": "Исследование рынка",   "assignee_id": "usr_alice", "project_ids": ["proj_rnd_q2"] },
  { "title": "Прототип решения",    "assignee_id": "usr_bob",   "project_ids": ["proj_rnd_q2"] },
  { "title": "Отчёт по результатам", "assignee_id": "usr_alice", "project_ids": ["proj_rnd_q2"] }
]
